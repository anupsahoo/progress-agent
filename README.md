# Progress Agent — Sports2000 Modernized

**Live demo:** https://anupsahoo.github.io/progress-agent/

A live React website showing Progress OpenEdge ABL programs from the `sports2000` demo database, modernized by [FlowPilot AI](https://github.com/anupsahoo/flowpilot-ai).

Each screen shows the **original character-based Progress UI** on the left and the **modernized React UI** on the right — same data, same business logic, modern interface.

## Modules

| Original (.p) | Modernized Screen | Purpose |
|---------------|-------------------|---------|
| `customer_maint.p` | Customer Maintenance | Edit a customer record |
| `cust_orders.p` | Customer Orders | Lookup customer + their orders |
| `sample-customer-list.p` | Customer List | Sortable/filterable customer report |
| `sample-order-report.p` | Order Report | Order details with line items |
| `sample-update-credit.p` | Credit Update | Batch credit limit adjustment |

## Data

All data comes from the `sports2000` sample database:
- `Customer` — 12 records
- `Order` — 12 records
- `OrderLine` — 17 records
- `Item` — 16 records

Extracted from the original `.d` pipe-delimited files and rendered via React.

## Running locally

```bash
npm install
npm run dev
```

## Deploying

Every push to `main` triggers the GitHub Actions workflow in `.github/workflows/deploy-pages.yml` which builds the Vite app and publishes to GitHub Pages.
